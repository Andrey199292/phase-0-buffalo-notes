# Чуть более продвинутая работа с гитом

## Работа с гитом в процессе решения задачи:

1. `git checkout -b yourBranchName` – задаём имя нашей ветки
2. Пишем код, решающий задачу
3. Коммитим все свои изменения
4. `git checkout main`
5. `git pull` – подтягиванием данные с main- в нашу ветку
6. Возвращаемся в свою ветку git checkout yourBranchName
7. `git merge main`
8. Если есть конфликты – решаем их
9. `git push –set-upstream origin yourBranchName` - пушим репозиторий: с привязкой к main

(P.S – можно сперва прописать git push, появится ошибка, предложит выше написанный путь; следующей командой можем спокойно его скопировать, если трудно запомнить на первое время)

10. Создаём пулл реквест

## Базовые команды:

- Работа с коммитами и удалёнными репозиториями:
    1. `git add filenames` – добавляет файлы для коммита
    3. `git comit -m 'message'` – коммит добавленные файлы с сообщением
    4. `git fetch` – запрашивает информацию о ветках с удалённого репозитория
    5. `git push` – отправляет данные на удалённый репозиторий
    6. `git pull` – получает данные с удаленного репозитория

- Работа с ветками:
    1. `git checkout name` – переключаемся на существующую ветку
    2. `git checkout -b name` – создаем новую ветку с именем name и переключаемся на неё
    3. `git branch` – получаем список доступных веток
    4. `git merge name` – запускает процесс слияния двух веток (чаще всего, слияние будет с main)
    5. `git merge –continue` – продолжает процесс слияния после разрешения мердж-конфликтов
    6. `git merge –abort` – сбрасывает слияние клеток
